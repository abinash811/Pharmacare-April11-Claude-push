PharmaCare — Refinement Audit

Specific issues found in codebase · April 2025 · References exact files + line patterns

P1 Critical — Consistency breaks the system feel
Two parallel button systems
AppButton.jsx and ui/button.jsx both exist and are used in different pages. They look nearly identical but have different APIs, sizes, and hover behaviour. AuthPage uses shadcn Button; most other pages use AppButton. This will drift over time.
AuthPage.js uses <Button> from ui/button.jsx → h-9 default
All pages should use AppButton — or unify AppButton into ui/button.jsx variants
Deprecate AppButton. Merge its 5 variants into buttonVariants in ui/button.jsx. Same API, one file.
Font mismatch on BillingWorkspace
The billing workspace hardcodes BillingWorkspace/index.jsx line: style={{ fontFamily: 'Manrope, sans-serif' }} The rest of the app uses IBM Plex Sans. Manrope is only meant as a display/heading font. This makes the most-used screen feel visually different.
Remove inline fontFamily. BillingWorkspace should inherit body font (IBM Plex Sans) like every other page.
MetricCard hardcodes Tailwind colors instead of design tokens
Dashboard/components/MetricCard.jsx uses bg-green-50 text-green-600 directly, bypassing the brand token system. When the brand color changes, metric cards won't update.
const COLOR_CLASSES = { green: 'bg-green-50 text-green-600' }
Use semantic vars: bg-[var(--status-green-bg)] text-[var(--status-green-text)]
Replace hardcoded Tailwind color strings with CSS var references. Keeps token system as single source of truth.
P2 Important — UX gaps vs international standard
MetricCard title hierarchy is flat — value and label share one line
MetricCard.jsx puts title + subtitle in one <p> separated by a bullet •. International dashboards (Shopify, Stripe, Linear) always give value, label, and change separate hierarchy levels.
<p>{title} • {subtitle}</p>
<p class="text-2xl font-bold">{value}</p><p class="text-xs text-muted">{title}</p><p class="text-xs text-muted">{subtitle}</p>
Separate into 3 distinct elements: value (bold, 24px), label (12px muted), change (11px green/red). Clear visual scanning.
Keyboard shortcuts are undiscoverable
BillingWorkspace/index.jsx has excellent shortcuts (Ctrl+F, F8, F12, Ctrl+B) but they are completely invisible to users. No tooltip, no help panel, no keyboard shortcut indicator on buttons. Power users never discover them.
Add ⌨ keyboard hint badges to relevant buttons (e.g. "Save & Print F12", "Hold F8"). A small kbd-shortcut tooltip on hover is the international standard (see Notion, Linear, Figma).
iPad: tap targets below 44px on nav items and table rows
Nav items use h-9 (36px) which is fine for desktop but fails the WCAG 2.5.5 touch target minimum of 44px on iPad. Table action buttons ("View", "Print") are even smaller.
On iPad (below md breakpoint), apply h-11 (44px) to nav items via Tailwind: h-9 md:h-9 pattern. Table action buttons need min-h-[44px] on touch devices.
No loading skeleton on BillingList / Inventory — only Dashboard has it
Dashboard has a proper animate-pulse skeleton loader. Other pages (Billing, Inventory, Purchases) show nothing while data loads — just an empty table. Inconsistent loading UX.
Use the existing shared/TableSkeleton.jsx component (already built!) on all list pages. It's imported in 0 pages currently.
P3 Polish — the details that make it feel premium
Auth page gradient is the only gradient in the app — feels disconnected
AuthPage.js: linear-gradient(135deg, #667eea 0%, #764ba2 100%) — purple/blue gradient that has zero connection to the Steel Blue brand palette. First impression of the product doesn't match the product.
Replace with brand-aligned background: solid #1a2332 (sidebar color) or a very subtle brand-tint (#f0f7ff) with the logo mark as a watermark. Ties login to the product identity.
No empty state on Dashboard when data is zero
If a new pharmacy has zero sales, metric cards show "₹0" and charts render empty axes. International products show an onboarding nudge: "Create your first bill to see data here."
Add zero-state check in Dashboard: if metrics.total_sales === 0, show a welcome card with quick-start actions (New Bill, Add Inventory, Add Supplier).
index.css and App.css both define body font — one overrides the other
index.css sets system fonts on body. App.css sets IBM Plex Sans. Load order determines which wins. Fragile.
Remove font-family from index.css body. Keep only App.css as the font authority. One file, one source.
TableSkeleton.jsx exists but is used nowhere
components/shared/TableSkeleton.jsx — a complete, well-built skeleton component. Zero imports found across the codebase. Dead code that should be the loading standard.
Import and use in BillingOperations, PurchasesList, Customers, Suppliers, InventorySearch. Estimated: ~5 files, ~3 lines each.
Summary: 3 P1 fixes (font, button unification, token consistency) · 4 P2 UX gaps (hierarchy, shortcuts, touch targets, skeletons) · 4 P3 polish items.
Biggest ROI: fix the font mismatch on BillingWorkspace + use TableSkeleton everywhere. 30 minutes of work, large perceived quality jump.
